# Azərbaycan dilində proqram təminatı mühəndisliyi terminləri
Proqram təminatı mühəndisliyi terminlərinin Azərbaycan dilindəki qarşılıqlarını tapırıq.

[//]: # (-----------------------------------------------------------------------------------------------------------------------------------)
# A

Abstraction - Abstraksiya: Mürəkkəb sistemlərin sadələşdirilməsi və əsas xüsusiyyətlərə fokuslanma prosesi.

Access Control - Giriş Nəzarəti: Sistemə və məlumatlara kimin giriş edə biləcəyini müəyyən edən siyasətlər və prosedurlar.

Access Modifier - Giriş Modifikatoru: Sinif və obyektlərə giriş hüquqlarını tənzimləyən proqramlaşdırma dilindəki xüsusiyyət.

Active Directory - Aktiv Kataloq: Microsoft sistemlərində istifadəçi və resursları idarə etmək üçün mərkəzi məlumat bazası.

Adapter Pattern - Adapter Nümunəsi: Müxtəlif interfeysləri uyğunlaşdıran və onları bir-birinə qoşan dizayn nümunəsi.

Agile Methodology - Çevik Metodologiya: Tez-tez dəyişən tələblərə cavab verən, iterativ və sıx əməkdaşlıq tələb edən proqram inkişaf metodologiyası.

Agile Software Development - Çevik Proqram Təminatı İnkişafı: Proqram təminatının çevik və iterativ inkişafı, istifadəçi tələblərinə uyğunlaşma.

Algorithm - Alqoritm: Müəyyən bir problemi həll etmək üçün ardıcıl addımların və qaydaların dəstəsi.

Align - Ahəngdə olmaq: Müxtəlif hissələri və ya komponentləri uyğunlaşdırmaq və birgə işləməsini təmin etmək.

Alpha Testing - Alfa Testi: Tətbiqin ilkin versiyasının daxili istifadəçilər tərəfindən test edilməsi.

Allocation - Ayrılma (Təyin etmə): Resursların və ya yaddaşın istifadə üçün ayrılması.

Analysis - Təhlil: Məlumatların və proseslərin sistematik araşdırılması.

Analytics - Analitika: Məlumatların təhlil edilməsi və onlardan mənalı nəticələr çıxarılması.

API (Application Programming Interface) - API (Tətbiq Proqramlaşdırma İnterfeysi): Proqramlar arasında məlumat mübadiləsini təmin edən interfeys.

Application - Tətbiq: İstifadəçilərə müəyyən bir məqsəd üçün xidmət edən proqram.

Application Lifecycle Management (ALM) - Tətbiq Həyat Dövrü İdarəetməsi: Tətbiqin inkişafı, yerləşdirilməsi və saxlanılması proseslərini idarə etmək.

Application Server - Tətbiq Serveri: Tətbiqlərin işləməsini təmin edən və istifadəçilərə xidmət göstərən server.

Architecture - Memarlıq: Sistemin strukturu və komponentlərinin necə təşkil edildiyini göstərən plan.

Artifact - Artefakt: Proqram inkişafı və ya test prosesi zamanı yaradılan hər hansı bir məhsul və ya sənəd.

Asynchronous Programming - Asinxron Proqramlaşdırma: Proqramın bir neçə əməliyyatı paralel yerinə yetirərək, birinin tamamlanmasını gözləmədən digərlərini icra etməsi.

Assembly Language - Yığın Dili: Maşın dilinə yaxın, aşağı səviyyəli proqramlaşdırma dili.

Assert - Təmin Etmək: Proqramın müəyyən şərtlərə uyğun işlədiyini yoxlamaq üçün istifadə olunan əməliyyat.

Assertion - Təminat: Proqramın müəyyən şərtlərlə işlədiyini yoxlayan bir növ test.

Assessment - Qiymətləndirmə: Proqramın və ya sistemin performansının və ya keyfiyyətinin ölçülməsi.

Assignment - Tapşırıq: Proqramlaşdırma dilində verilən və ya təyin edilmiş dəyərin dəyişkənə təyin edilməsi.

Association - Assosiasiya: Siniflər arasında əlaqə və ya münasibət.

Asynchronous Communication - Asinxron Əlaqə: Məlumat mübadiləsinin vaxt sinxronizasiyasına ehtiyac duymadan aparılması.

Atomic Operation - Atomik Əməliyyat: Bölünməz və tamamilə yerinə yetirilən əməliyyat.

Authentication - Doğrulama: İstifadəçinin kimliyini təsdiqləyən proses.

Authorization - Səlahiyyətləndirmə: İstifadəçiyə müəyyən resurslara və əməliyyatlara girişi təsdiqləmək.

Automation - Avtomatlaşdırma: Təkrarlanan və ya əməliyyatların əl ilə yerinə yetirilməsi əvəzinə avtomatik şəkildə həyata keçirilməsi.

Automated Testing - Avtomatlaşdırılmış Test: Proqram təminatının sınaqdan keçirilməsi üçün avtomatik alətlərin istifadəsi.

Availability - Əlçatanlıq: Sistemin və ya tətbiqin istifadəçilər üçün davamlı işləmə vəziyyəti.

Audit Trail - Auditin İzi: Sistemdə edilən bütün əməliyyatların və dəyişikliklərin qeydiyyatı və izlənməsi.

# B

Back-End - Arxa Tərəf: Sistemin istifadəçilərə görünməyən, amma məlumatların işlənməsi və saxlanması üçün cavabdeh olan hissəsi.

Back-End Framework - Arxa Tərəf Çərçivəsi: Arxa tərəf proqramlaşdırma üçün istifadə olunan struktur və alətlər toplusu.

Backlog - Arxa Plan: İcra olunması gözlənilən işlərin, tapşırıqların və ya xüsusiyyətlərin siyahısı.

Backward Compatibility - Geriyə Uyumluluq: Yeni bir versiyanın köhnə versiyalarla uyğun olaraq işləməsi xüsusiyyəti.

Balanced Scorecard - Balanslı Kart: İş performansını müxtəlif sahələrdə ölçmək üçün istifadə olunan bir idarəetmə aləti.

Balance Sheet - Balans Hesabatı: Bir şirkətin maliyyə vəziyyətini göstərən hesabat.

Base Class - Əsas Sinif: Digər siniflərin miras aldığı və onların strukturlarını müəyyən edən sinif.

Base Method - Əsas Metod: Bir sinifdə müəyyən edilmiş və alt siniflər tərəfindən istifadə edilə bilən metod.

Basic Block - Əsas Blok: Proqramın icra edilə bilən ən kiçik hissəsi, ardıcıl şəkildə işlənir.

Behavioral Model - Davranış Modeli: Sistemin və ya komponentin davranışlarını təsvir edən model.

Best practices - ən yaxşı üsullar, ən yaxşı metodlar: Ən yaxşı nəticəni əldə etmək üçün qəbul edilmiş və tətbiq olunan metodologiyalar.

Big-O Notation - Big-O Notasiyası: Bir alqoritmin performansını ölçmək üçün istifadə olunan və onun asimptotik davranışını ifadə edən notasiya.

Binary - İkilik: İki simvol (0 və 1) ilə təmsil olunan məlumatların işlənməsi.

Binary Search - İkilik Axtarışı: Sıralanmış verilənlərdə bir elementi tapmaq üçün istifadə olunan alqoritm.

Binary Tree - İkilik Ağacı: Hər düyünün yalnız iki uşağa sahib olduğu ağac quruluşu.

Binding - Bağlama: Funksiyalar və verilənlər arasında əlaqə yaratma prosesi.

Bit - Bit: İkilik sistemində ən kiçik məlumat vahidi, 0 və ya 1.

Block - Blok: Kod və ya məlumatların qruplaşdırıldığı və müəyyən bir funksiyanı yerinə yetirən hissə.

Bootstrap - Bootstrap: Tətbiq və veb səhifələrin sürətli inkişafı üçün açıq mənbəli çərçivə.

Bottleneck - Boğulma: Sistemdə resursların tıxanması və ya tələbin gecikməsinə səbəb olan hissə.

Boundary Value Analysis (BVA)- Hüdud Dəyəri Təhlili: Test üsulu, giriş dəyərlərinin sərhədlərində səhvləri tapmaq üçün istifadə olunur.

Branch - budağ: Proqramın və ya kodun paralel inkişafını təmin edən hər hansı bir yeni yola yönəltmə.

Breakpoint - Dayanma Nöqtəsi: Proqramın icrasını müəyyən bir nöqtədə dayandıran səhv ayıklama vasitəsi.

Browser Compatibility - Brauzer Uyumluluğu: Veb saytın müxtəlif brauzerlərdə düzgün işləməsi.

Build - Quruluş: Proqramın hazırlanması, tərtib edilməsi və sınaqdan keçirilməsi prosesi.

Build Automation - Quruluş Avtomatlaşdırılması: Quruluş və test proseslərinin avtomatikləşdirilməsi.

Build Script - Quruluş Skripti: Proqramın tərtib edilməsi və qurulması üçün yazılmış skript.

Bulk Data Transfer - Kütləvi Məlumat Köçürməsi: Böyük həcmdə məlumatın bir sistemdən digərinə ötürülməsi.

Bus - Yuva: Kompüterin müxtəlif komponentləri arasında məlumat ötürülməsi üçün istifadə edilən şəbəkə.

Business Analysis - Biznes Təhlili: Biznesin və ya layihənin tələblərini və ehtiyaclarını təhlil etmək və qiymətləndirmək.

Business Logic - Biznes Məntiqi: Tətbiqdə və ya sistemdə biznes qaydalarını və əməliyyatlarını həyata keçirən məntiq.

Business Model - Biznes Modeli: Bir biznesin necə fəaliyyət göstərəcəyi və gəlir əldə edəcəyinə dair plan.

Business Process Management (BPM) - Biznes Proseslərinin İdarə Edilməsi: Biznes proseslərini optimallaşdırmaq və idarə etmək üçün istifadə olunan alətlər və metodlar.

Byte - Bayt: Kompüterdə məlumatların təmsil edilməsi üçün istifadə olunan əsas ölçü vahidi (8 bit).

Bytecode - Bayt Kodu: Yüksək səviyyəli proqramların aşağı səviyyəli maşın koduna çevrilən orta səviyyəli təmsilatı.

Buffer - Yaddaş Tamponu: Məlumatların müvəqqəti saxlanması üçün istifadə edilən yaddaş sahəsi.

Bug - Xəta: Proqramın düzgün işləməməsinə səbəb olan səhv və ya nasazlıq.

# C

Cache - Keş: Məlumatları sürətli əldə etmək məqsədilə müvəqqəti saxlayan yaddaş sahəsi.

Call Stack - Çağırış Yığını: Proqramın icrasını izləmək və idarə etmək üçün istifadə olunan verilənlər strukturu.

Callback - Çağırış: Bir funksiyanın digər funksiyaya ötürülərək icra edildikdən sonra geri çağırılması prosesi.

Camel Case - Kərbəz Üslubu: Bir çox sözün birləşdirilməsi ilə yaradılan dəyişkən adlandırma üsulu, hər yeni sözün böyük hərflə başlaması.

Canonical Model - Kanonik Model: Məlumatların və ya sistemlərin sadələşdirilmiş və ümumi bir təmsil forması.

Cardinality - Kardinalik: Verilənlər bazasında əlaqələr arasındakı məlumatın çoxluğu.

Case Study - Hal Araşdırması: Müəyyən bir hadisə və ya problemə dair dərindən təhlil və araşdırma.

Catch Block - İstisna hallarln idarəetmə bloku: Proqramdakı xətaları tutmaq üçün istifadə olunan kod bloku, əsasən istisnaları idarə etmək üçün.

Causal Loop - Səbəb-Dövran: Bir hadisənin digərinə təsir etdiyi və geri təsir dövrünün əmələ gəldiyi model.

Class - Sinif: Obyektlərin yaradılmasında istifadə olunan şablon və ya plan.

Class Diagram - Sinif Diaqramı: Obyektlər və siniflər arasındakı əlaqələri göstərən UML diaqramı.

CLI (Command Line Interface) - Komanda Xətti İnterfeysi: İstifadəçilərin əmrləri daxil edərək proqramlarla qarşılıqlı əlaqə qurduğu interfeys.

Cloud Computing - Bulud Hesablama: İnternet üzərindən verilənlər və proqram təminatının təmin edilməsi.

Cohesion - Koheziya: Sinif və ya modulların daxilindəki əlaqənin güclülüyü, yaxud bir-birinə bağlı olma dərəcəsi.

Collaborative Software - Əməkdaşlıq Proqramı: Bir çox istifadəçinin eyni vaxtda və ya birgə işlərlə bağlı məlumatları paylaşmasını təmin edən proqramlar.

Collection - Kolleksiya: Məlumatları saxlamaq və idarə etmək üçün istifadə olunan verilənlər tipi, məsələn, massivlər və siyahılar.

Command Pattern - Əmrlər Nümunəsi: Bir əmri obyekt kimi təsvir edən və icra etməyə imkan verən dizayn nümunəsi.

Commitment - Öhdəlik: Proqram inkişaf etdiricilərinin və ya komandasının layihəyə verdiyi zaman və resurs öhdəliyi.

Commit - Təsdiqləmək: Yaradılan dəyişiklikləri versiya idarəetmə sisteminə daxil etmək.

Compiler - Tərtibçi: Mənbə kodunu daha aşağı səviyyəli və ya icra edilə bilən koda çevirən proqram.

Composition - Kompozisiya: Obyektlər arasında əlaqəni təmin edən və bir obyektin başqa bir obyektin içindəki funksiyaları istifadə etməsinə imkan verən dizayn yanaşması.

Component - Komponent: Bir sistemin və ya tətbiqin əsas hissəsi və ya modulunu təşkil edən bir vahid.

Component-Based Software Engineering (CBSE) - Komponent əsaslı Proqram mühəndisliyi: Proqramın müxtəlif komponentlərdən birləşdirilərək hazırlanması yanaşması.

Concurrency - Paralellik: Eyni vaxtda bir neçə prosesin işlədilməsi.

Concurrency Control - Paralellik Nəzarəti: Paralel işləyən proseslərin və məlumatların idarə edilməsi, sinxronizasiya edilməsi.

Condition - Şərt: Proqramın qərar vermə prosesində istifadə olunan vəziyyət.

Configuration - Konfiqurasiya: Sistem və ya proqramın parametrlərinin və xüsusiyyətlərinin təyin edilməsi.

Configuration Management - Konfiqurasiya İdarəetməsi: Proqram və sistemin dəyişikliklərinin izlənməsi, idarə edilməsi və saxlanılması prosesi.

Confidentiality - Məxfilik: Məlumatların yalnız yetkili şəxslərə açıq olması təminatı.

Constructor - Konstruktor: Obyektin yaradılmasını təmin edən xüsusi metod.

Container - Konteyner: Məlumatların və obyektlərin saxlanılması və idarə olunması üçün istifadə olunan strukturlar.

Continuous Integration (CI) - Davamlı İntegrasiya: Tətbiqdəki bütün kod dəyişikliklərinin avtomatik olaraq test edilməsi və inteqrasiya olunması.

Control Flow - Nəzarət Axını: Proqramın icra axınını təyin edən strukturlar və təlimatlar.

Controller - Nəzarətçi: Model-View-Controller (MVC) arxitekturasında verilənləri idarə edən və istifadəçi əməliyyatlarını qəbul edən komponent.

Coupling - Qoşulma: Sistem və ya komponentlər arasında əlaqənin gücü; aşağı qoşulma daha az əlaqə və daha çox müstəqillik deməkdir.

Coverage - Əhatə: Testin proqramın neçə faizini əhatə etdiyini göstərən göstərici.

Critical Path - Əsas Yol: Proyektin tamamlanması üçün tələb olunan minimum vaxtı müəyyən edən fəaliyyətlər ardıcıllığı.

Cross-Platform - Çarpaz Platforma: Müxtəlif əməliyyat sistemlərində və cihazlarda işləyə bilən proqram təminatının inkişaf etdirilməsi.

CRUD (Create, Read, Update, Delete) - CRUD (Yarat, Oxu, Yenilə, Sil): Proqramların əsas funksionallıqlarını əhatə edən əməliyyatlar.

Cryptography - Kriptoqrafiya: Məlumatların təhlükəsizliyini təmin etmək məqsədilə şifrələmə və deşifrləmə texnologiyaları.

Customer Relationship Management (CRM) - Müştəri Əlaqələri İdarəetməsi: Müştəri məlumatlarını və əlaqələrini idarə edən sistemlər və yanaşmalar.

Custom Software - Xüsusi Proqram: Müəyyən bir istifadəçi və ya təşkilatın ehtiyaclarına uyğun olaraq yaradılmış proqram.

Cyclomatic Complexity - Siklomatika Çətinliyi: Alqoritminin nə qədər mürəkkəb olduğunu və onun test edilməsinin nə qədər çətin olduğunu ölçən göstərici.

# D

Data Access Layer (DAL) - Məlumat Giriş Qatı: Məlumatların sistemə daxil olması və sistemdən çıxması üçün interfeys təmin edən proqram hissəsi.

Data Access Object (DAO) - Məlumat Giriş Obyekti: Məlumatların sistemlər arasında ötürülməsi və əldə edilməsi üçün istifadə olunan obyekt.

Data Analytics - Məlumat Analitikası: Böyük məlumat dəstlərinin təhlili və bu məlumatlardan nəticə çıxarılması prosesi.

Data Binding - Məlumat Bağlama: Verilənlərin istifadəçi interfeysi elementləri ilə birləşdirilməsi prosesi.

Data Dictionary - Məlumat Lüğəti: Məlumatların strukturlarını və onların əlaqələrini təsvir edən sənəd və ya sistem.

Data Flow Diagram (DFD) - Məlumat Axını Diaqramı: Məlumatların bir sistemdə necə axdığını və işlənildiyini göstərən vizual diaqram.

Data Integrity - Məlumatın Doğruluğu: Məlumatların tam və dəqiq olmasını təmin edən texnologiyalar və prinsiplər.

Data Migration - Məlumat Köçürülməsi: Məlumatların bir sistemdən digərinə köçürülməsi prosesi.

Data Modeling - Məlumat Modelləşdirmə: Məlumatların strukturunu təmsil edən və onları idarə etməyə kömək edən modelin yaradılması.

Data Structure - Məlumat Strukturu: Məlumatları səmərəli saxlamaq və işləmək üçün təşkil edilmiş verilənlər forması.

Data Synchronization - Məlumat Sinxronizasiyası: Fərqli sistemlər və verilənlər bazalarında məlumatların uyumlu və eyni olmasını təmin edən proses.

Data Warehouse - Məlumat Anbarı: Böyük həcmdə məlumatın saxlandığı və analiz edildiyi mərkəzi verilənlər bazası.

Database - Verilənlər Bazası: Məlumatların saxlandığı və idarə olunduğu təşkil olunmuş sistem.

Database Management System (DBMS) - Verilənlər Bazası İdarəetmə Sistemi: Verilənlər bazalarını yaratmaq, idarə etmək və istifadə etmək üçün istifadə olunan proqram təminatı.

Database Schema - Verilənlər Bazası Şeması: Verilənlər bazasında cədvəllərin, sütunların və əlaqələrin necə təşkil olunduğunu göstərən struktur.

Datalog - Dataloq: Məlumatların və əməliyyatların işlənməsini və saxlanılmasını təmin edən məlumat bazası dilidir.

Debugging - Xəta Aradan Qaldırma: Proqramdakı səhvlərin və nasazlıqların tapılması və düzəldilməsi prosesi.

Decentralized - Mərkəzləşdirilməmiş: Sistem və ya tətbiqin mərkəzi idarəetməyə ehtiyac olmadan fəaliyyət göstərməsi.

Decision Table - Qərar Cədvəli: Müxtəlif şərtlərə görə alınması lazım olan qərarları göstərən cədvəl.

Declarative Programming - Deklarativ Proqramlaşdırma: Proqramın nə etməli olduğunu təsvir edən, amma necə etməli olduğunu göstərməyən proqramlaşdırma üsulu.

Defect - Xəta: Proqramda və ya sistemdə gözlənilməyən davranış və ya nasazlıq.

Defect Density - Xətaların Sıxlığı: Müəyyən bir kod və ya sistem sahəsindəki xətaların nisbəti.

Defragmentation - Defraqmentasiya: Diskdəki məlumatların daha səmərəli saxlanması üçün parçalanmış faylların birləşdirilməsi.

Delegation - Təqdim etmə: Bir işin və ya funksiyanın başqa birinə verilməsi.

Dependency Injection - Asılılıq Vurulması: Bir obyektin digər obyektlərə olan asılılıqlarının xaricdən təmin edilməsi.

Deploy - yerləşdirmək: Tətbiq və ya sistemin istifadəçilər üçün aktivləşdirilməsi.

Deployment - Yerləşdirmə: Proqram və ya sistemin hazırlanmasından sonra istehsala yerləşdirilməsi.

Deployment Pipeline - Yerləşdirmə Borusu: Tətbiqin inkişafdan istehsal mühitinə qədər hər bir mərhələdə avtomatik olaraq inteqrasiya və yerləşdirilməsini təmin edən proses.

IDE (Integrated Development Environment) - İnteqrasiya Olunmuş İnkişaf Mühiti: Proqram yazmaq üçün istifadə olunan alətlər və proqramların birləşməsi, məsələn, kod redaktoru, tərtibçi və səhv ayıklayıcı.

Identity Management - Kimlik İdarəetməsi: İstifadəçi kimliklərini idarə etmək və onların doğruluğunu təmin etmək üçün istifadə olunan alətlər və proseslər.

I/O (Input/Output) - Giriş/Çıxış: Proqramların və cihazların məlumat mübadiləsi etməsini təmin edən əməliyyatlar və proseslər.

IaaS (Infrastructure as a Service) - İnfrastruktur Xidmət Kimi: Fiziki resursların (məsələn, serverlər, yaddaş, şəbəkə) onlayn olaraq təmin edilməsi və istifadəçilərə bulud əsaslı olaraq təqdim edilməsi xidməti.

Icon - İkonka: Bir tətbiq və ya sistemin funksiyalarını vizual şəkildə təmsil edən qrafik təsvir.

Identifier - Tanıdıcı: Proqramlaşdırma dilində dəyişkənlər, funksiyalar və obyektlər kimi obyektləri təmsil edən unikal ad.

Incremental Development - Artımlı İnkişaf: Layihənin və ya proqramın funksionallığının tədricən, addım-addım inkişaf etdirilmə prosesi.

Induction - İnduksiya: Ümumi nəticələrə və ya qaydalara çatmaq üçün xüsusi nümunələrdən istifadə edən təhlil üsulu.

Inheritance - İrsi: Obyekt yönümlü proqramlaşdırmada, bir sinifin başqa sinifdən xüsusiyyətləri və metodları miras alması prosesi.

Initialization - İlkinləşdirmə: Bir dəyişkənin və ya obyektin başlanğıc dəyəri ilə təyin edilməsi prosesi.

Instance - Nümunə: Sinifdən yaradılan obyekt və ya konkret bir hal.

Instruction - Təlimat: Proqramın və ya sistemin hansı əməliyyatı yerinə yetirməli olduğunu göstərən kod və ya komanda.

Integration - İntegrasiya: Müxtəlif komponentlərin və ya sistemlərin birlikdə işləməsini təmin etmək üçün birləşdirilməsi prosesi.

Integration Testing - İntegrasiya Testi: Sistemin müxtəlif hissələrinin birlikdə işləməsini yoxlamaq üçün həyata keçirilən test növü.

Interface - İnterfeys: Müxtəlif proqram və ya sistem komponentləri arasında qarşılıqlı əlaqəni təmin edən metod və ya protokol.

Internet of Things (IoT) - Əşyaların İnterneti: Şəbəkəyə qoşulmuş cihazların və obyektlərin bir-biri ilə əlaqə qurması və məlumat mübadiləsi etməsi texnologiyası.

Isolation - İzolyasiya: Sistemin və ya komponentlərin müstəqil şəkildə işləməsini təmin etmək üçün digər hissələrdən ayrılması prosesi.

Instance Variable - Nümunə Dəyişəni: Bir sinifdə obyektin hər bir nümunəsi üçün saxlanılan və hər nümunənin özünə aid olan dəyişən.

Iteration - Təkrarlama: Bir əməliyyatın və ya prosesin bir neçə dəfə ardıcıl olaraq təkrarlanması.

Iterator - İterator: Məlumatların üzərində ardıcıl gəzinti etməyə imkan verən obyekt və ya interfeys.

Impact Analysis - Təsir Təhlili: Proqram və ya sistemdəki dəyişikliklərin digər komponentlər və ya sistemlər üzərində yaratacağı təsirin qiymətləndirilməsi.

Input - Giriş: Sistemdə verilənlər və ya məlumatların daxil edilməsi prosesi.

Instance Method - Nümunə Metodu: Sinifdə müəyyən edilmiş və obyektin nümunələrinə aid olan metod.

Intelligent System - Ağıllı Sistem: İnsan müdaxiləsi olmadan müəyyən bir məqsəd üçün müstəqil şəkildə işləyə bilən sistemlər.

Integration Pattern - İntegrasiya Nümunəsi: Müxtəlif sistem və tətbiqlərin birləşdirilməsini asanlaşdıran ümumi yanaşmalar və strukturlar.

Implementation - Həyata Keçirmə: Proqramın və ya sistemin gerçəkləşdirilməsi və icra olunması.

Input Validation - Giriş Yoxlanması: İstifadəçi tərəfindən daxil edilən məlumatların düzgünlüyünü və etibarlılığını yoxlama prosesi.

Indexing - İndeksləmə: Məlumatları tapmaq və sürətli şəkildə əldə etmək üçün verilənlər bazasında məlumatları təşkil etmə üsulu.

# J

JavaBeans - JavaBeans: Java proqramlaşdırma dilində komponentlər üçün istifadə olunan bir model, xüsusilə verilənlər bazası və istifadəçi interfeysi ilə əlaqə qurma məqsədilə.

JDBC (Java Database Connectivity) - JDBC (Java Verilənlər Bazası Bağlantısı): Java tətbiqlərinin verilənlər bazasına qoşulmasına imkan verən API.

JIT (Just-In-Time Compilation) - JIT (Vaxtında Tərcümə): Proqramın icra zamanı tərcümə edilərək optimallaşdırılmasını təmin edən bir texnologiya.

JSON (JavaScript Object Notation) - JSON (JavaScript Obyekt Qeydiyyatı): Məlumat mübadiləsi üçün istifadə olunan yüngül və oxunaqlı məlumat formatı.

JVM (Java Virtual Machine) - JVM (Java Virtual Maşın): Java proqramlarını müxtəlif əməliyyat sistemlərində işlətmək üçün lazım olan virtual maşın.

Job Scheduling - İş Cədvəlləşdirmə: Sistem və ya proqram daxilindəki vəzifələrin müəyyən zaman kəsiyində avtomatik olaraq həyata keçirilməsini təmin edən proses.

JAR (Java ARchive) - JAR (Java ARxiv): Java proqramlarını paketləşdirmək və yaymaq üçün istifadə olunan arxiv formatı.

Javadoc - Javadoc: Java dilində yazılmış proqram kodunun sənədləşdirilməsi üçün istifadə olunan vasitə.

# K

Kanban - Kanban: Proqram inkişafı və ya digər iş prosesləri üçün işlərin vizual olaraq izlənməsini və idarə edilməsini təmin edən metodologiya.

Kernel - Nüvə: Əməliyyat sisteminin əsas hissəsi, hardware ilə proqram təminatı arasındakı əlaqəni təmin edir və əsas funksiyaları yerinə yetirir.

Key Performance Indicator (KPI) - Əsas Performans Göstəricisi: Təşkilatın və ya sistemin effektivliyini ölçmək üçün istifadə olunan göstəricilər.

KLOC (Kilo Lines of Code) - Kilo Kod Sətiri: Proqramın ölçüsünü qiymətləndirmək üçün istifadə olunan ölçü vahidi, adətən 1,000 sətir kod olaraq qəbul edilir.

Knowledge Base - Bilik Bazası: Təşkilatda mövcud biliklərin saxlanıldığı və istifadəçilərin məlumat əldə etməsinə kömək edən sistem.

KISS (Keep It Simple, Stupid) - KISS (Sadə saxla, axmaq): Proqram inkişafında və dizaynda sadəlik və təkrarlamağa qarşı çıxma prinsipi.

Kernel Mode - Nüvə Rejimi: Əməliyyat sisteminin və hardware-in tam nəzarətində olan və yüksək prioritetli əməliyyatların icra edildiyi rejim.

Knowledge Management - Bilik İdarəetməsi: Təşkilatın biliklərini toplamaq, saxlamaq və paylaşmaq üçün sistemlər və proseslər.

Keep-alive - Bağlantını Açıq Saxlama: Şəbəkə üzərində əlaqə saxlanılmasına imkan verən mexanizm, xüsusən müştəri-server qarşılıqlı əlaqələrində istifadə olunur.

KVM (Kernel-based Virtual Machine) - KVM (Nüvə əsaslı Virtual Maşın): Linux əməliyyat sistemində virtualizasiya təmin edən və virtual maşınların idarə edilməsini təmin edən texnologiya.

Knowledge Worker - Bilik İşçisi: Əsasən məlumatları işləyən və təşkilatda bilikləri idarə edən şəxslər.

# L

Lambda Function - Lambda Funksiyası: Proqramlaşdırma dillərində adlandırılmamış, kiçik və birbaşa istifadə oluna bilən funksiyalar.

Latency - Latenlik: Məlumatın bir nöqtədən digərinə çatma müddəti, əsasən şəbəkə və ya sistemin cavab vermə sürəti.

Layer - Qat: Proqram və ya sistemin müxtəlif funksionallıqlarını təcrid edərək təşkil edən və müəyyən məqsədlər üçün istifadə olunan strukturlar.

Learning Curve - Öyrənmə Əyri: Yeni bir bacarıq və ya prosesin öyrənilmə sürətini və çətinliyini göstərən qrafik.

Legacy System - Köhnə Sistem: Vaxtı keçmiş və texniki cəhətdən müasir tələblərə cavab verməyən, lakin hələ də istifadə olunan sistem.

Library - Kitabxana: Təkrarlanan kod və funksionallığı təmin edən, digər proqramlar və ya sistemlər tərəfindən istifadə edilə bilən proqram təminatı modulları.

Lifecycle - Həyat Dövrü: Proqramın və ya sistemin yaradılmasından istifadəyə verilməsinə qədər olan bütün mərhələləri əhatə edən dövr.

Lightweight - Yüngül: Aşağı resurs istehlakı və sürətli işləmə qabiliyyətinə malik olan sistem və ya proqram.

Line of Code (LOC) - Kod Sətiri (LOC): Proqramın və ya skriptin yazılmış sətirlərinin ümumi sayı.

Load Balancing - Yük Paylanması: Sistemdəki əməliyyat yükünü bir neçə server və ya resurs arasında bərabər şəkildə bölüşdürmə prosesi.

Local Variable - Lokal Dəyişkən: Yalnız müəyyən bir funksiyada və ya metodda istifadə olunan dəyişkən.

Lock - Kilid: Bir resursun və ya məlumatın eyni anda yalnız bir istifadəçi tərəfindən dəyişdirilməsini təmin edən mexanizm.

Log - Loq: Proqramın fəaliyyətini izləmək və təhlil etmək üçün qeydə alınan məlumatların toplusu.

Loop - Dönmə: Bir və ya daha çox əməliyyatın müəyyən şərtlərlə təkrarlanması.

Low-Level Language - Aşağı Səviyyəli Dil: Maşın dilinə yaxın və birbaşa prosessor tərəfindən icra edilə bilən proqramlaşdırma dilləri.

Low-Code Development - Aşağı Kod İnkişafı: Minimal kod yazaraq proqram və tətbiqlər inkişaf etdirməyə imkan verən yanaşma.

Liskov Substitution Principle - Liskov Əvəzləmə Prinsipi: Obyekt yönümlü proqramlaşdırmada, bir sinifin alt siniflərinin, özlərinə aid olmayan siniflərlə əvəzlənə bilməsi prinsipidir.

Liveness - Canlılıq: Sistem və ya tətbiqin fəaliyyətini davam etdirmək və işləyən vəziyyətdə qalmasını təmin etmək üçün işləmə vəziyyəti.

Load Testing - Yük Testi: Proqramın və ya sistemin müxtəlif yük və iş yükü altında necə performans göstərdiyini yoxlamaq üçün həyata keçirilən test növü.

Loop Unrolling - Dönmə Açıqlığı: Performansı artırmaq məqsədilə dövrün (loop) müəyyən hissələrinin əl ilə açılması və ya optimallaşdırılması.

Logging - Loqlaşdırma: Proqramın fəaliyyətini izləmək və təhlil etmək məqsədilə məlumatların qeyd edilməsi.

Linear Search - Xətti Axtarış: Məlumatları sıralanmamış bir şəkildə bir-bir yoxlayaraq tapmağa çalışan axtarış alqoritmi.

Liveness Property - Canlılıq Xüsusiyyəti: Bir sistemin və ya prosesin işini davam etdirmə qabiliyyəti, məsələn, proqramın qısa zamanda nəticə verməsi.

# M

Machine Learning - Maşın Öyrənməsi: Kompüterlərin və sistemlərin verilənlərdən öyrənərək, müəyyən nəticələr çıxartma və qərar vermə qabiliyyəti.

Mainframe - Əsas Çərçivə: Böyük miqyasda verilənlər işləmə və işləmə gücünə malik olan, adətən iri müəssisələrdə istifadə edilən kompüterlər.

Malware - Zərərli Proqram: Sistemi və ya şəbəkəni pozmaq məqsədilə hazırlanmış proqram təminatı.

Mapping - Xəritələmə: Məlumatları müəyyən bir struktura və ya formatda təmsil etmə prosesi.

Markup Language - İşarələmə Dili: Məlumatların və ya mətnlərin müəyyən bir strukturla təmsil edilməsi üçün istifadə olunan dil (məsələn, HTML, XML).

Maintainability - Saxlanıla Bilərlik: Proqramın və ya sistemin asanlıqla təmir edilməsi və inkişaf etdirilməsi qabiliyyəti.

Management Information System (MIS) - İdarəetmə Məlumat Sistemi: Bir təşkilatın idarəetmə qərarlarını dəstəkləmək üçün məlumatları toplayan və təhlil edən sistem.

Manual Testing - Əl ilə Testləşdirmə: Proqramın düzgünlüyünü yoxlamaq üçün avtomatlaşdırılmamış, insan müdaxiləsi ilə həyata keçirilən test.

Microservices - Mikroservislər: Böyük və kompleks tətbiqlərin kiçik, müstəqil xidmətlərə bölünərək inkişaf etdirilməsi yanaşması.

Middleware - Aralıq Proqram: Müxtəlif proqramların və ya sistemlərin bir-biri ilə qarşılıqlı əlaqə qurmasını təmin edən proqram qatıdır.

Model-View-Controller (MVC) - Model-Görünüş-Nəzarətçi: Tətbiq arxitekturasında məlumat, istifadəçi interfeysi və istifadəçi əməliyyatlarının ayrılmasını təmin edən dizayn nümunəsi.

Modularity - Modulyarlıq: Proqramın və ya sistemin müstəqil işləyə bilən hissələrə ayrılması, bu da inkişaf və saxlanma prosesini asanlaşdırır.

Monitor - Monitor: Proqramın və ya sistemin fəaliyyətini izləyən və məlumatları toplayan alət.

Memory Management - Yaddaşın İdarə Edilməsi: Proqramın işləmə zamanı yaddaş resurslarını idarə edən və optimallaşdıran proses.

Modeling - Modelləşdirmə: Bir sistemin və ya prosesin təmsilini yaradaraq onun davranışını və xüsusiyyətlərini öyrənmək və təhlil etmək.

Mocking - Mocking (Təqlid Etmə): Test zamanı real obyektlər və ya komponentlər əvəzinə, onların təqlidlərini istifadə edərək funksionallığın yoxlanması.

Multithreading - Çox İplilik: Eyni anda bir neçə işin və ya prosesin paralel olaraq icra edilməsi.

Mutex (Mutual Exclusion) - Mutex (Mütəşəkkil İstifadə): Yalnız bir işin və ya prosesin bir vaxtda müəyyən bir resursu istifadə etməsini təmin edən mexanizm.

Migration - Köçürmə: Məlumatların və ya sistemlərin bir mühitdən digərinə keçirilməsi.

Message Queue - Mesaj Növbəsi: Asinxron olaraq məlumatların göndərildiyi və qəbul edildiyi bir növbə sistemi, proqramların arasında məlumat mübadiləsini asanlaşdırır.

Minimum Viable Product (MVP) - Minimum Yetərli Məhsul: Bir məhsulun bazara təqdim olunmadan əvvəl ən əsas xüsusiyyətləri ilə hazırlanmış ilkin versiyası.

Modeling Language - Modelləşdirmə Dili: Sistemləri və ya prosesləri vizual olaraq təsvir etmək üçün istifadə olunan dil və ya alətlər (məsələn, UML).

Merging - Birleştirmə: İki və ya daha çox kod şaxəsinin birləşdirilməsi, hər birinin dəyişikliklərinin uyğunlaşdırılması.

Machine - Maşın: Proqramın işlədiyi və əməliyyatların yerinə yetirildiyi fiziki və ya virtual cihaz.

Memory Leak - Yaddaş Sızması: Proqramın işləməsi zamanı istifadə olunan yaddaşın sərbəst buraxılmaması və zamanla yaddaşın dolması problemi.

Multithreading - Çox İplilik: Eyni anda bir neçə işin və ya prosesin paralel olaraq icra edilməsi.

# N

Naming Convention - Adlandırma Qaydası: Proqramlaşdırma dillərində dəyişkənlər, funksiyalar, siniflər və digər obyektlərin adlarını müəyyən bir qaydada yazma üsulu.

Namespace - Ad Məkânı: Həmçinin ad sahəsi kimi də bilinir, eyni adda olan elementlərin qarşısını almaq və təşkil etmək üçün istifadə olunan struktural təşkilat.

Natural Language Processing (NLP) - Təbii Dilin Emalı (NLP): Kompüterlərin insan dilini anlama və təhlil etmə qabiliyyəti.

Navigation - Naviqasiya: İstifadəçinin tətbiq və ya veb saytı arasında hərəkət etməsini təmin edən əməliyyatlar və sistemlər.

NFR (Non-Functional Requirement) - NFQ (Funksional Olmayan Tələb): Bir sistemin işləməsi ilə bağlı performans, təhlükəsizlik və digər qeyri-funksional tələbləri təyin edən xüsusiyyətlər.

Normalization - Normalizasiya: Verilənlər bazasında məlumatların təkrarlanmasını və redundansını aradan qaldırmaq üçün strukturlaşdırma prosesi.

Node - Nöqtə: Şəbəkədə və ya verilənlər bazasında məlumatların və əlaqələrin yerləşdiyi əsas birim.

Non-Relational Database - Əlaqəsiz Verilənlər Bazası: Məlumatları cədvəllər şəklində deyil, daha sərbəst strukturda saxlayan verilənlər bazası növü.

Null - Null: Dəyəri olmayan və ya mövcud olmayan obyekt və ya dəyişkən vəziyyəti.

N-tier Architecture - N-Qatlı Arxitektura: Tətbiq və ya sistemin funksionallığının müxtəlif mərhələlərə bölünməsi, hər mərhələnin ayrı-ayrı qatlar olaraq təşkil edilməsi.

Non-Volatile Memory - Qeyri-qiymətli Yaddaş: Elektrik kəsildikdə belə məlumatları saxlayan yaddaş növü, məsələn, flash yaddaş və ya ROM.

Noise - Səs-Küylü: Məlumatın və ya verilənlərin içərisində yanlış və ya lazımsız məlumatların olması vəziyyəti.

Normalization (Database) - Normalizasiya (Verilənlər Bazası): Verilənlər bazasında təkrarlanmanın qarşısını almaq və məlumatların inteqrasiya olunmasını təmin etmək üçün strukturlaşdırma prosesi.

Network Protocol - Şəbəkə Protokolu: Şəbəkə üzərindən məlumatların ötürülməsi və alqoritm əlaqələrinin təyin edilməsi üçün istifadə olunan qaydalar toplusu.

NoSQL - NoSQL: Ənənəvi əlaqəli verilənlər bazalarına alternativ olaraq, daha elastik və genişlənə bilən verilənlər bazası sistemləri, məsələn, MongoDB və Cassandra.

Notification - Xəbərdarlıq: Sistem və ya tətbiq tərəfindən istifadəçiyə təqdim edilən xəbərdarlıq və ya məlumat.

Nondeterministic - Deterministik Olmayan: Eyni şəraitdə belə hər dəfə fərqli nəticələr verə bilən proses.

Neural Network - Neyron Şəbəkəsi: Məlumatları təhlil etmək və öyrənmək üçün istifadə olunan, beynin iş prinsipinə əsaslanan bir maşın öyrənmə modelidir.

Network - Şəbəkə: Kompüterlər və cihazlar arasında məlumat mübadiləsini təmin edən qurğular və əlaqələr.

No-Operation (NOP) - Heç bir Əməliyyat: Sistemdə heç bir dəyişiklik etməyən əməliyyat növü, ümumiyyətlə test və ya asanlaşdırıcı kodlarda istifadə olunur.

# O

Object - Obyekt: Obyekt yönümlü proqramlaşdırma dilində, məlumatları və bu məlumatlarla əlaqəli əməliyyatları saxlayan struktur.

Object-Oriented Programming (OOP) - Obyekt Yönümlü Proqramlaşdırma (OYP): Proqram inkişafında obyektlər və onların qarşılıqlı əlaqələri üzərində qurulmuş yanaşma.

Object Relational Mapping (ORM) - Obyekt Əlaqəli Xəritələmə: Obyekt yönümlü proqramlaşdırma dilləri ilə əlaqəli verilənlər bazası arasında əlaqə qurma metodu.

Optimization - Optimizasiya: Proqramın və ya sistemin performansını artırmaq məqsədilə, qaynaqların daha səmərəli istifadəsi.

Open Source - Açıq Mənbə: Proqram təminatının kodlarının sərbəst şəkildə paylaşılması və istifadəçilər tərəfindən dəyişdirilməsi.

Operating System (OS) - Əməliyyat Sistemi: Kompüterin hardware və proqram təminatını idarə edən və istifadəçi əməliyyatlarına cavab verən sistem proqramı.

Operation - Əməliyyat: Proqramda verilənlər üzərində yerinə yetirilən hərəkət və ya hesablamalar.

Overloading - Yükləmə: Obyekt yönümlü proqramlaşdırmada, eyni adla bir neçə funksiyanın və ya metodun yaradılması.

Overriding - Üstələmə: Alt sinifin, valideyn sinifindəki metodun davranışını dəyişdirməsi prosesi.

Orchestration - Orkestrasiya: Müxtəlif xidmət və sistemlərin vahid məqsəd üçün uyğun şəkildə işləməsini təmin edən proses.

Overflow - Daşma: Proqramda və ya sistemdə verilənlərin qəbul edilə bilən həddən çox olması nəticəsində məlumat itkisi və ya səhv yaranması.

Ongoing Development - Davamlı İnkişaf: Proqram və ya sistemin istifadəyə verildikdən sonra da inkişaf etdirilməsi və təkmilləşdirilməsi prosesi.

Outsourcing - Xaricə Verilməsi: Proqram inkişafı və digər iş funksiyalarının üçüncü tərəf şirkətlərinə və ya mütəxəssislərə həvalə edilməsi.

Object Graph - Obyekt Qrafiki: Obyektlər arasındakı əlaqələri və onların strukturlarını vizual şəkildə təmsil edən qrafik.

OpenAPI - Açıq API: Sistemin digər tətbiqlər və xidmətlərlə qarşılıqlı əlaqə qurmasına imkan verən və açıq şəkildə təklif olunan API interfeysi.

Offline - Offline: İnternetə və ya şəbəkəyə qoşulmadan işləyən sistem və ya proqram.

On-demand - Tələbə uyğun: Xidmətlərin və ya resursların istifadəçi tələblərinə əsasən təqdim edilməsi.

Opaque Data - Qaranlıq Məlumat: Məlumatların məzmununun istifadəçilər və ya digər sistemlər tərəfindən görünməyən, yalnız müəyyən funksiyalar tərəfindən işlənən məlumat.

OpenID - OpenID: İnternet üzərində istifadəçi identifikasiyasını təmin etmək üçün istifadə olunan açıq protokol.

Operating Environment - Əməliyyat Mühiti: Proqram və ya sistemin işlədiyi və qarşılıqlı əlaqədə olduğu mühit.

Opt-in - İcazə Verilməsi: İstifadəçiyə müəyyən məlumatların toplanması və ya xidmətlərin təqdim edilməsi üçün icazə verilən seçim.

Opt-out - Çıxmaq: İstifadəçiyə məlumatların toplanmasından və ya xidmətlərdən istifadə etməkdən imtina etmək seçimi.

# P

Pagination - Səhifələmə: Böyük məlumat dəstlərinin bir neçə hissəyə bölünməsi və hər bir hissənin ayrılıqda təqdim edilməsi.

Parallel Computing - Paralel Hesablama: Hesablama işlərinin eyni vaxtda bir neçə prosessor və ya kompüter tərəfindən yerinə yetirilməsi.

Parameterized Testing - Parametrləşdirilmiş Test: Testin müxtəlif parametrlərlə bir neçə dəfə icra edilməsi ilə müxtəlif nəticələr əldə etmək üçün istifadə olunan test yanaşması.

Patch - Təmir: Proqram təminatında mövcud səhvləri düzəltmək üçün tətbiq edilən kiçik dəyişiklik.

Performance Testing - Performans Testi: Sistemin və ya proqramın sürətini, fəaliyyətini və dayanıqlığını ölçmək üçün həyata keçirilən test.

Pipeline - Boru Kəməri: Proqram və ya tətbiqin inkişafından istehsala qədər olan mərhələlərdə avtomatlaşdırılmış iş axını.

Platform - Platforma: Proqramların işlədiyi əməliyyat mühiti, o cümlədən hardware və software komponentləri.

Polling - Sorğu Vermə: Proqramın və ya sistemin müəyyən bir vaxt intervallarında server və ya resurslarla əlaqə quraraq məlumatları əldə etməsi prosesi.

Portability - Portativlik: Proqramın və ya sistemin müxtəlif mühitlərdə və cihazlarda işləyə bilməsi qabiliyyəti.

Postmortem - Postmortem: Bir layihə və ya fəaliyyətin bitməsindən sonra baş verən problemlərin və səhvlərin təhlil edilməsi prosesi.

Push Notification - İtələmə Bildirişi: Mobil tətbiq və ya veb tətbiq vasitəsilə istifadəçiyə yönləndirilən xəbərdarlıqlar.

Proxy - Proxy: Başqa bir sistem və ya server ilə əlaqə qurarkən, aradakı əlaqə nöqtəsi olaraq istifadə olunan alət.

Process - Proses: Proqram və ya əməliyyatın yerinə yetirilməsi üçün müəyyən edilmiş addımlar və qaydalar.

Procedure - Prosedur: Müəyyən edilmiş məqsədə çatmaq üçün izlənilən və ya yerinə yetirilən bir sıra əməliyyatlar.

Product Backlog - Məhsul Arxası: Agile inkişaf metodologiyasına əsasən, məhsulun inkişaf mərhələlərini təyin edən prioritetləşdirilmiş tapşırıqlar siyahısı.

Project Management - Layihə İdarəetməsi: Layihənin məqsədinə çatmaq üçün resursların və tapşırıqların idarə edilməsi prosesi.

Prototyping - Prototipləşdirmə: Bir məhsulun və ya sistemin ilkin versiyasını hazırlayaraq istifadəçi rəyini almaq məqsədilə edilən inkişaf mərhələsi.

Push - İtələmək: Məlumatların bir sistemdən digərinə və ya istifadəçiyə göndərilməsi prosesi.

Pseudocode - Psevdo-kod: Proqramın hərəkətlərini yazılı olaraq izah edən, amma təbii dilə yaxın bir dil.

Public Key Infrastructure (PKI) - İctimai Açar İnfrastrukturu: Şifrələmə və təhlükəsizlik protokolları təmin edən, ictimai və özəl açarlardan istifadə edən sistem.

Pull Request - Çəkiliş Tələbi: Proqramın inkişafında bir kod dəyişikliyinin əsas koda inteqrasiya edilməsi üçün təklifin təqdim edilməsi.

Prototype - Prototip: Bir sistemin və ya məhsulun ilkin modelini nümayiş etdirən nümunə.

Public Method - İctimai Metod: Hər kəsin istifadə edə biləcəyi və ya çağıracağı metod, obyektin daxilindən və xaricindən istifadə oluna bilər.

Performance Bottleneck - Performans Boğulması: Bir proqram və ya sistemin performansını məhdudlaşdıran və inkişaf etdirilməli olan hissə.

Permission - İcazə: Sistemdə və ya tətbiqdə istifadəçilərə verilən müxtəlif hüquq və imkanlar.

# Q

Quality Assurance (QA) - Keyfiyyət Təminatı: Proqram təminatının keyfiyyətini artırmaq məqsədilə həyata keçirilən bütün fəaliyyətlər və prosedurlar.

Quality Control (QC) - Keyfiyyət Nəzarəti: Proqramın və ya məhsulun müəyyən edilmiş keyfiyyət standartlarına uyğunluğunu yoxlamaq üçün həyata keçirilən test və yoxlama fəaliyyəti.

Query - Sorğu: Verilənlər bazasında məlumatları əldə etmək üçün istifadə olunan sorğu dili (məsələn, SQL).

Queue - Növbə: Məlumatların sıralanmış şəkildə saxlanması və növbə ilə işlənməsi, ilk daxil olanın ilk çıxması prinsipi (FIFO).

Quicksort - QuickSort: Effektiv sıralama alqoritmi, verilənləri nisbətən kiçik hissələrə bölərək sürətli şəkildə sıralamağa imkan verir.

Quantization - Kvantlaşdırma: Analog məlumatları rəqəmsal formata çevirmək prosesi, məlumatın kəsirli hissələrinin təmsil edilməsini təmin edir.

Quorum - Kvorum: Bir sistemin və ya təşkilatın fəaliyyətə başlaması və ya qərar qəbul etməsi üçün tələb olunan minimum iştirakçı sayı.

Qualitative Analysis - Keyfiyyət Təhlili: Verilənlərin və ya məlumatların ölçülə bilməyən və ya mütləq rəqəmsal olmayan aspektlərini təhlil etmək.

Quick Response (QR) - Sürətli Cavab (QR): Məlumatın və ya sistemin tez və effektiv şəkildə cavab verməsini təmin edən xüsusiyyət.

Questionnaire - Sorğu Forması: İstifadəçi rəyləri toplamaq və məlumat əldə etmək üçün istifadə olunan bir alət və ya form.

Quantum Computing - Kvant Hesablama: Kvant mexanikası prinsiplərinə əsaslanan və daha sürətli məlumat emalı təmin edən yeni texnologiya.

Qualitative Data - Keyfiyyətli Məlumat: Anketlər və digər metodlarla əldə edilən, ölçülə bilməyən və ya təsvir edilən məlumat növü.

# R

Race Condition - Yarış Şərti: Bir proqramın və ya sistemin nəticələrinin zamanlama və sıralama fərqləri səbəbindən gözlənilməyən şəkildə dəyişməsi vəziyyəti.

Refactoring - Refaktorinq: Proqramın xarici davranışını dəyişdirmədən kodunun strukturunu və təşkilini yaxşılaşdırmaq prosesi.

Regression Testing - Regresyon Testi: Sistemdə və ya tətbiqdəki yeni dəyişikliklər və ya əlavələrin əvvəlki funksionallıqları pozmadığını yoxlamaq üçün həyata keçirilən test növü.

REST (Representational State Transfer) - REST (Təmsili Dövlət Transferi): Web xidmətlərinin sadə, dövlətə əsaslanan və asan inteqrasiya edilə bilən bir protokolu.

Reliability - Etibarlılıq: Bir proqramın və ya sistemin müəyyən müddət ərzində düzgün işləmək qabiliyyəti.

Redundancy - Redundantlıq: Sistemdə və ya şəbəkədə əlavə ehtiyat komponentlərin yerləşdirilməsi, yəni, bir komponentin işləməməsi halında başqa birinin fəaliyyətini təmin etmək.

Risk Management - Risk İdarəetməsi: Potensial riskləri müəyyən etmək, qiymətləndirmək və onların qarşısını almaq və ya təsirlərini minimuma endirmək üçün həyata keçirilən tədbirlər.

Reusability - Təkrar İstifadə: Bir kod və ya komponentin başqa tətbiqlərdə və ya layihələrdə təkrar istifadə edilməsi qabiliyyəti.

Repository - Anbar: Məlumatların və ya kodun saxlandığı və idarə edildiyi yer, məsələn, versiya idarəetmə sistemlərində.

Request for Comments (RFC) - Şərh Tələb Edilən Təklif: Proqram təminatı və ya şəbəkə protokolları üçün müəyyən bir standartın və ya təklifin ictimai şəkildə müzakirə olunması və qəbul edilməsi prosesidir.

Release Management - Buraxılış İdarəetməsi: Proqram və ya sistemin yeni versiyalarını və yeniliklərini planlaşdırmaq, inkişaf etdirmək və yayımlamaq prosesi.

Runtime - İcra Vaxtı: Proqramın işlədiyi müddət, adətən proqramın başlanğıcından sonuna qədərki zaman.

RDF (Resource Description Framework) - RDF (Resurs Təsvir Çərçivəsi): Veb məlumatlarını təmsil etmək üçün istifadə olunan açıq standart, xüsusilə semantik veb üçün.

Release Candidate (RC) - Buraxılış Namizədi: Sistem və ya proqram təminatının son versiyaya yaxın olan, lakin bəzi kiçik testlərdən keçməsi hələ tələb olunan versiya.

Repository Pattern - Anbar Nümunəsi: Məlumatları verilənlər bazası və ya digər saxlanılan strukturlardan almaq və təmin etmək üçün istifadə olunan dizayn nümunəsi.

Robustness - Dayanıqlılıq: Bir proqramın və ya sistemin gözlənilməyən və ya hətta səhv şəraitlərdə də işləməsini təmin edən xüsusiyyət.

Reflexive Method - Refleksiv Metod: Bir obyektin və ya sinifin öz xüsusiyyətlərini və davranışlarını özündən sorğulama metodudur.

Remote Procedure Call (RPC) - Uzaq Prosedur Çağırışı: Bir tətbiqin başqa bir maşında işləyən xidmətə prosedur və ya metod çağırışı göndərməsi.

Routing - Yönləndirmə: Şəbəkə və ya web tətbiqində istifadəçinin tələblərini düzgün yerə yönləndirmək üçün istifadə olunan metod.

Reusability - Təkrar İstifadə: Proqramın və ya komponentlərin müxtəlif tətbiqlərdə təkrar istifadə edilə bilməsi qabiliyyəti.

Read-Only - Yalnız Oxuma: Məlumatların yalnız oxunması və dəyişdirilməməsi üçün təyin edilmiş vəziyyət.

Resource Allocation - Resursların Təyin Edilməsi: Proqram və ya sistemdə mövcud olan resursların, məsələn, yaddaşın, prosessor gücünün və ya şəbəkə bant genişliyinin düzgün şəkildə bölüşdürülməsi.

Release Note - Buraxılış Qeydləri: Tətbiqin və ya proqramın yeni versiyası ilə əlaqəli əhəmiyyətli dəyişikliklər və xüsusiyyətlər barədə istifadəçilərə verilən məlumat.

Real-Time Processing - Reallaşdırılmış Vaxtda Emal: Verilənlərin dərhal emal edilməsi və nəticələrin ani olaraq təqdim edilməsi tələb olunan sistemlər.

Recursive Function - Rekursiv Funksiya: Özünü təkrarlayan və problemi daha kiçik hissələrə ayıraraq həll edən funksiya.

Role-Based Access Control (RBAC) - Rol Əsaslı Giriş Nəzarəti (RBAC): İstifadəçilərin müəyyən rola uyğun olaraq resurslara və verilənlərə nəzarət etməsini təmin edən sistem.

# S

Scrum - Scrum: Agile metodologiyasına əsaslanan, komandaların müntəzəm iterasiyalarla məhsul inkişaf etdirməsini təmin edən bir layihə idarəetmə çərçivəsi.

Search Engine Optimization (SEO) - Axtarış Motoru Optimizasiyası (SEO): Veb saytlarının axtarış motorlarında daha yüksək sıralarda yer alması üçün edilən optimallaşdırma işləri.

Security - Təhlükəsizlik: Proqram və sistemlərin məlumatların qorunması, məlumat sızmasının qarşısını almaq və hücumlardan qorunmaq məqsədilə dizayn və tədbirlər.

Semaphore - Semafor: Paralel proqramlaşdırmada bir prosesin digərini sinxronlaşdırmasına və qarşılıqlı resurs istifadəsini təmin etməsinə imkan verən mexanizm.

Server - Server: Məlumatları və xidmətləri şəbəkə üzərindən digər cihazlara təmin edən kompüter və ya proqram.

Service-Oriented Architecture (SOA) - Xidmətə Yönəlik Arxitektura (SOA): Proqram təminatının xidmətlərdən ibarət strukturda təşkil edilməsi və xidmətlərin bir-biri ilə qarşılıqlı əlaqəyə girməsi yanaşması.

Session - Sessiya: İstifadəçi və sistem arasındakı əlaqənin müəyyən bir müddət ərzində davam etməsi və verilənlər mübadiləsi.

Single Responsibility Principle (SRP) - Tək Məsuliyyət Prinsipi (SRP): Obyekt Yönümlü Proqramlaşdırma prinsipi, hər bir sinifin yalnız bir məsuliyyətinin olması və yalnız bir səbəbdən dəyişməsi lazım olduğunu bildirir.

Simulation - Simulyasiya: Proqram və ya sistemin davranışının başqa bir mühitdə və ya vəziyyətdə təkrarlanması.

Source Code - Mənbə Kodu: Proqramın işləməsini təmin edən yazılı qaydalar və təlimatlar toplusu, adətən bir proqramlaşdırma dilində yazılır.

Software Architecture - Proqram Arxitekturası: Proqramın və ya sistemin əsas strukturu, komponentləri və onların qarşılıqlı əlaqələrini təyin edən dizayn.

Software Development Life Cycle (SDLC) - Proqram Təminatı İnkişafının Həyat Dövrü (SDLC): Proqramın inkişafı üçün izlənilən mərhələlərdən ibarət metodologiya, məsələn, analiz, dizayn, inkişaf, test və tətbiq.

Software Engineering - Proqram Mühəndisliyi: Proqram təminatının hazırlanması, tətbiqi, idarə edilməsi və saxlanılması üçün mühəndislik prinsiplərinin və texnologiyalarının tətbiqi.

Software Maintenance - Proqramın Saxlanması: Proqramın ilkin istifadəyə verildikdən sonra həyata keçirilən dəyişikliklər, təkmilləşdirmələr və səhv düzəltmələri.

Software Requirements - Proqram Tələbləri: Müəyyən bir proqram və ya sistemin işləməsi üçün tələb olunan xüsusiyyətlər və funksionallıqlar.

Software Testing - Proqram Testi: Proqramın düzgün işləməsini və istifadəçi tələblərinə cavab verməsini təmin etmək məqsədilə həyata keçirilən yoxlama prosesi.

SQL (Structured Query Language) - SQL (Strukturlu Sorğu Dili): Verilənlər bazasında məlumatları idarə etmək, sorğulamaq və işləmək üçün istifadə olunan dildir.

State Machine - Vəziyyət Maşını: Müəyyən bir girişə əsaslanaraq sistemin müxtəlif vəziyyətlərə keçməsini təmin edən model.

Static Analysis - Statik Təhlil: Proqramın icrası olmadan, yalnız mənbə kodu əsasında səhvləri və potensial zəiflikləri aşkar etmək üçün həyata keçirilən təhlil.

Static Binding - Statik Bağlama: Metod və ya funksiyanın icra anında deyil, tərtib zamanı əlaqələndirilməsi prosesi.

System Architecture - Sistem Arxitekturası: Bir sistemin əsas strukturu, komponentləri və onların qarşılıqlı əlaqələrini təyin edən dizayn.

System Integration - Sistem İnteqrasiyası: Müxtəlif sistem və komponentlərin birləşdirilərək bütöv bir sistem halına gətirilməsi prosesi.

System Testing - Sistem Testi: Proqramın və ya sistemin bütöv işləməsini və bütün komponentlərin birgə işləməsini yoxlamaq məqsədilə həyata keçirilən test növü.

Service - Xidmət: Şəbəkə üzərində və ya tətbiq daxilində müəyyən funksionallığı təmin edən modul və ya proqram təminatı.

Singleton Pattern - Singleton Nümunəsi: Hər bir tətbiqdə yalnız bir nümunənin mövcud olduğu və bu nümunənin təkrarlanmaması üçün nəzərdə tutulan dizayn nümunəsi.

Synchronization - Sinxronizasiya: Eyni vaxtda bir neçə prosesin və ya işin resurslardan düzgün və təhlükəsiz istifadə etməsini təmin edən mexanizm.

Software Quality - Proqram Keyfiyyəti: Proqramın gözlənilən nəticələri verdiyi və müştəri tələblərinə cavab verdiyi keyfiyyət səviyyəsi.

Semantic Versioning - Semantik Versiyalaşdırma: Proqram təminatının versiyalarını tərtib etmək üçün istifadə olunan metodologiya, kiçik dəyişikliklər, yeni funksiyalar və böyük yeniliklərə görə versiya nömrələrini müəyyən edir.

Scalability Testing - Ölçeklənə Bilərlik Testi: Sistemin müxtəlif ölçülərdə necə performans göstərdiyini yoxlamaq məqsədilə həyata keçirilən test növü.

Soft Skills - Yumşaq Bacarıqlar: Proqram mühəndisliyində, xüsusilə komanda işi və müştəri ilə əlaqə qurmaq üçün tələb olunan şəxsi bacarıqlar.

# T

Task - Tapşırıq: Sistem və ya proqramda yerinə yetirilməsi lazım olan fəaliyyət və ya əməliyyat.

Test - Test: Proqramın və ya sistemin işləməsini və funksionallığını yoxlamaq üçün həyata keçirilən yoxlama prosesi.

Test Case - Test Halı: Proqramın müəyyən bir xüsusiyyətinin və ya funksiyasının yoxlanması üçün yaradılmış test addımlarının və gözlənilən nəticələrin təsvir edildiyi sənəd.

Test Environment - Test Mühiti: Testlərin həyata keçirilməsi üçün istifadə olunan proqram və ya sistemin fiziki və ya virtual mühiti.

Test Plan - Test Planı: Testlərdə izləniləcək strategiya, məqsəd və metodları müəyyən edən sənəd.

Test Scenario - Test Ssenarisi: Test prosesində yerinə yetiriləcək bir neçə addımı birləşdirən və müəyyən bir funksiyanın və ya xüsusiyyətin davranışını yoxlayan testin ümumi təsviri.

Test Suite - Test Paketləri: Bir neçə əlaqəli test halını və ssenarisini bir yerdə toplayan birləşmiş testlər topluluğu.

Test Driven Development (TDD) - Test Yönümlü İnkişaf (TYYİ): Proqram inkişafında əvvəlcə testlərin yazılması və sonra həmin testlərə uyğun kodun yazılması metodologiyası.

Testing - Testləşdirmə: Proqramın düzgünlüyünü və keyfiyyətini yoxlamaq üçün həyata keçirilən hər hansı bir yoxlama və təhlil prosesi.

Thread - İplik: Eyni vaxtda bir neçə əməliyyatın paralel olaraq işlədiyi, proqramın icrasını idarə edən ən kiçik işləyən vahid.

Throughput - Keçiricilik: Bir sistemin müəyyən bir zaman aralığında işləyə bildiyi məlumat və ya əməliyyat həcmi.

Time Complexity - Vaxt Çətinliyi: Alqoritmin işləməsi üçün lazım olan vaxtı göstərən ölçü, verilənlərin ölçüsünə bağlı olaraq dəyişir.

Token - Token: Bir proqram dilində və ya protokolda istifadə olunan kiçik, mənalı vahid, məsələn, açar sözlər və ya əməliyyat simvolları.

Tracer - İzçi: Proqramın və ya sistemin müxtəlif mərhələlərində baş verən fəaliyyətləri izləyən və qeydə alan alət.

Transaction - Transaksiya: Bir və ya bir neçə əməliyyatın birləşdirildiyi, tam və düzgün şəkildə yerinə yetirilməsi lazım olan bir vahid əməliyyat.

Transcoding - Transkoding: Bir formatdan digərinə məlumat çevirmə prosesi, məsələn, video və ya audio fayllarının çevrilməsi.

Testing Framework - Test Çərçivəsi: Testləri avtomatlaşdırmağa və idarə etməyə imkan verən alətlər və kitabxanalar toplusu.

Test Automation - Testin Avtomatlaşdırılması: Testlərin avtomatik alətlər vasitəsilə həyata keçirilməsi prosesi.

Third-Party Software - Üçüncü Tərəf Proqramı: Birbaşa inkişaf etdiricinin nəzarətində olmayan və ya yaradılmayan proqramlar.

Total Quality Management (TQM) - Ümumi Keyfiyyət İdarəetməsi (ÜKI): Keyfiyyətin hərtərəfli idarə edilməsi üçün istifadə olunan yanaşma, həm məhsul, həm də xidmət keyfiyyətini artırmağa yönəlib.

Turing Test - Turing Testi: Bir maşının insan kimi düşünmə qabiliyyətini ölçmək üçün istifadə olunan test.

Typedef - Tip Təyini: Proqramlaşdırma dilində, xüsusi verilənlər tipi yaratmaq və ya dəyişdirmək üçün istifadə olunan komanda.

# U

UAT (User Acceptance Testing) - İstifadəçi Qəbul Testi: İstifadəçi tərəfindən həyata keçirilən və proqramın müştəri tələblərinə uyğunluğunu yoxlayan test növü.

UI (User Interface) - İstifadəçi İnterfeysi: İstifadəçi və sistem arasında qarşılıqlı əlaqəni təmin edən vizual komponentlər və dizayn.

UDP (User Datagram Protocol) - UDP (İstifadəçi Datagram Protokolu): Şəbəkə protokolu, məlumatların sürətli və az gecikmə ilə göndərilməsini təmin edir, amma etibarlılıq təmin etmir.

UML (Unified Modeling Language) - UML (Birləşdirilmiş Modelləşdirmə Dili): Proqramın və ya sistemin müxtəlif aspektlərini vizual olaraq təmsil etmək üçün istifadə olunan rəsmi dil və alətlər toplusu.

Unit Testing - Vahid Testi: Proqramın hər bir kiçik hissəsinin düzgün işlədiyini yoxlamaq üçün aparılan test növü.

Upgrade - Yeniləmə: Proqramın və ya sistemin yeni versiyasına keçid etmək və ya mövcud xüsusiyyətləri təkmilləşdirmək prosesi.

Usability - İstifadə Edilə Bilərlik: Sistemin və ya proqramın istifadəçilər üçün nə dərəcədə asan və rahat istifadə oluna bildiyini ölçən keyfiyyət.

Use Case - İstifadə Halı: İstifadəçinin sistemlə qarşılıqlı əlaqədə olduğu vəziyyətlərin və ssenarilərin təsviri.

User Story - İstifadəçi Hekayəsi: Agile metodologiyasına əsaslanan, istifadəçinin tələblərini və ya ehtiyaclarını sadə şəkildə təsvir edən qısa və ümumi izah.

UUID (Universally Unique Identifier) - UUID (Ümumi Unikal İdentifikator): Hər bir identifikatorun unikal olduğu və çox güman ki, təkrarlanmayacaq bir sistem tərəfindən təyin edilmiş dəyər.

Uptime - İşləmə Vaxtı: Proqramın və ya sistemin fasiləsiz işlədiyi müddət, istifadəçilər tərəfindən xidmətin əlçatan olduğu zaman.

Underfitting - Az Uyğunlaşdırma: Maşın öyrənmə alqoritmalarında modelin çox sadə olması və verilənlərdəki nümunələri yaxşı ümumiləşdirə bilməməsi vəziyyəti.

Upstream - Yuxarı Axın: Proqram təminatının və ya məlumatların bir sistemdə daha yüksək və ya ilkin mərhələlərə doğru hərəkət etməsi.

URL (Uniform Resource Locator) - URL (Birləşmiş Resurs Ünvanı): İnternet üzərindən bir qaynağa (məsələn, veb səhifəyə) ünvanlanan və onu tapmaq üçün istifadə olunan standart format.

Urgency - Təcili: Sistemdə və ya layihədə qarşılaşılan problemlərin və ya dəyişikliklərin həll edilməsinin nə dərəcədə təcili olduğunu müəyyən edən ölçü.

User Authentication - İstifadəçi Doğrulaması: Sistemdə və ya tətbiqdə istifadəçi identifikasiyasını və girişini təmin etmək üçün istifadə olunan metod və prosedurlar.

User Experience (UX) - İstifadəçi Təcrübəsi (UX): İstifadəçilərin sistemlə qarşılıqlı əlaqəsi zamanı əldə etdiyi ümumi təcrübə və məmnuniyyət.

User Interface Design - İstifadəçi İnterfeysi Dizaynı: İstifadəçilərin sistemlə qarşılıqlı əlaqədə olduğu vizual və interaktiv komponentlərin dizaynı.

Utility Software - Yaxşılaşdırıcı Proqram: Sistemin və ya istifadəçinin əməliyyatlarını asanlaşdırmaq və ya inkişaf etdirmək üçün istifadə olunan proqram təminatı (məsələn, antivirus proqramları).

Usage Metrics - İstifadə Ölçüləri: Sistemin və ya proqramın istifadəçilər tərəfindən necə istifadə edildiyini və nə qədər istifadə edildiyini ölçən göstəricilər.

# V

Validation - Doğrulama: Sistem və ya proqramın düzgün işləyib-işləmədiyini və ya müəyyən tələblərə cavab verib-vermədiyini yoxlama prosesi.

Value Object - Dəyər Obyekti: Obyekt yönümlü proqramlaşdırmada, heç bir unikal kimliyə malik olmayan, sadəcə dəyər daşıyan obyekt.

Version Control - Versiya Nəzarəti: Proqram təminatının inkişafı zamanı kodun müxtəlif versiyalarının izlənməsi və idarə edilməsi prosesidir.

Virtual Machine (VM) - Virtual Maşın (VM): Fiziki maşının resurslarını təqlid edən və fərqli əməliyyat sistemlərini işlətmək üçün istifadə olunan proqram və ya mühit.

Virtualization - Virtualizasiya: Fiziki resursların (məsələn, serverlər, yaddaş) bir neçə virtual mühitə bölünməsi, hər birinin müstəqil şəkildə işləməsini təmin edən texnologiya.

View - Görünüş: Model-Görünüş-Nəzarətçi (MVC) dizayn nümunəsində, məlumatların istifadəçiyə təqdim olunduğu vizual hissə.

Verification - Yoxlama: Sistemin və ya proqramın müəyyən bir xüsusiyyətinin və ya funksiyasının düzgün tətbiq olunduğunu yoxlama prosesi.

Vulnerability - Zəiflik: Bir proqramın və ya sistemin təhlükəsizliyini poza biləcək zəif nöqtə və ya təhlükə.

Versioning - Versiyalaşdırma: Proqram və ya sistemin müxtəlif inkişaf mərhələlərini və versiyalarını idarə etmək üçün tətbiq olunan metod.

Validation Testing - Doğrulama Testi: Proqramın və ya sistemin, müəyyən tələblərə və müştəri gözləntilərinə uyğunluğunu yoxlamaq üçün həyata keçirilən test.

Virtual Private Network (VPN) - Virtual Şəxsi Şəbəkə (VPN): İnternet üzərindən şifrəli əlaqə yaradaraq, uzaqdan təhlükəsiz əlaqə qurulmasını təmin edən şəbəkə texnologiyası.

ViewModel - Görünüş Modeli: MVC və MVVM dizayn nümunələrində istifadə olunan və görünüşün təqdim etdiyi məlumatları idarə edən model.

Vector - Vektor: Məlumatın sıralı və sıx bir strukturu, xüsusən verilənlər bazasında və proqramlaşdırma dillərində istifadə olunan bir növ verilənlər tipi.

Vagrant - Vagrant: Virtual mühitlər yaratmaq və idarə etmək üçün istifadə olunan açıq mənbə aləti.

Virtual Host - Virtual Host: Şəbəkədə bir serverin bir neçə domen və ya IP ünvanını eyni anda idarə etməsinə imkan verən texnologiya.

Velocity - Sürət: Agile inkişaf metodologiyasına əsasən, komandanın müəyyən bir vaxt ərzində tamamladığı işin ölçüsü.

Vault - Vault: Məlumatların təhlükəsiz şəkildə saxlanması və şifrələnməsi üçün istifadə olunan sistem və ya alət.

Vulnerability Assessment - Zəiflik Qiymətləndirməsi: Sistem və ya tətbiqin təhlükəsizlik zəifliklərini müəyyən etmək üçün həyata keçirilən təhlil prosesi.

Variable - Dəyişkən: Proqramlaşdırma dilində verilənlərin saxlandığı və dəyişdirilə bilən adlandırılmış obyekt.

Visitor Pattern - Ziyarətçi Nümunəsi: Objektdən ayrılmış kodları bir obyektin üzərində işləmək üçün istifadə edən dizayn nümunəsi.

# W

Web Application - Veb Tətbiqi: İstifadəçi ilə qarşılıqlı əlaqə qurmaq üçün internet üzərindən işləyən və brauzerlər vasitəsilə istifadə edilən proqram.

Web Services - Veb Xidmətləri: Müxtəlif sistemlər arasında məlumat mübadiləsi və qarşılıqlı əlaqə üçün istifadə olunan şəbəkə əsaslı xidmətlər.

Web Development - Veb İnkişafı: Veb saytların və veb tətbiqlərinin hazırlanması, dizaynı və proqramlaşdırılması prosesi.

White Box Testing - Ağ Qutu Testi: Proqramın daxili strukturunu və məntiqini test edən test növü, kodun hər hissəsini yoxlamağa yönəlir.

Wireframe - Tel Şəbəkəsi: Veb sayt və ya tətbiqin dizaynının ilkin və sadə vizual təqdimatı, strukturu və funksionallıqları göstərir.

Workflow - İş Axını: Məlumatın, tapşırıqların və ya proseslərin sistem və ya iş mühiti daxilində necə hərəkət etdiyini və idarə edildiyini göstərən model.

WYSIWYG (What You See Is What You Get) - WYSIWYG (Nə Görürsən, O Da Alırsan): Redaktor və ya proqramda yaradılan elementlərin nəzarət olunan təqdimatı ilə final məhsulun eyni olması prinsipi.

Web Framework - Veb Çərçivəsi: Veb tətbiqlərinin inkişafını asanlaşdıran, proqramçıların müxtəlif funksiyaları təkrarlamadan tətbiqləri yaratmalarına imkan verən alətlər və kitabxanalar toplusu.

WebSocket - WebSocket: İki tərəfli, tam-duplex şəbəkə əlaqəsi təmin edən protokol, veb tətbiqlərində real zamanlı məlumat mübadiləsini asanlaşdırır.

Workflow Automation - İş Axını Avtomatlaşdırması: Müəyyən iş proseslərini və tapşırıqları avtomatikləşdirmək üçün istifadə olunan alətlər və texnologiyalar.

Wi-Fi - Wi-Fi: Şəbəkələrdə məlumat ötürülməsi üçün simsiz əlaqə standartı.

Wrapper Class - Wrapper Sinifi: Verilənlər tipini obyektə çevirmək üçün istifadə olunan siniflər (məsələn, Java dilində Integer, Double sinifləri).

Web Crawling - Veb Axtarışı: Veb səhifələri tarayaraq məlumatları toplamaq və axtarış motorları üçün indeksləmək prosesi.

Workload - İş Yükü: Bir sistem və ya tətbiq tərəfindən işlənilən məlumat və ya icra olunan əməliyyatların həcmi.

# X

XaaS (Everything as a Service) - Hər Şey Xidmət Kimi: Müxtəlif xidmətlərin (məsələn, SaaS, PaaS, IaaS) internet üzərindən təmin edilməsi yanaşması.

XML (Extensible Markup Language) - XML (Genişlənə Bilən İşarələmə Dili): Məlumatların saxlanması və ötürülməsi üçün istifadə olunan, oxuna bilən və strukturlaşdırılmış şəbəkə dili.

XSS (Cross-Site Scripting) - XSS (Saytlararası Skript İcraatı): Veb tətbiqlərində təhlükəsizlik zəiflikləri vasitəsilə zərərli skriptlərin istifadəçi tərəfindən yüklənməsi və işlədilməsi.

XPath - XPath: XML sənədlərində məlumat axtarmaq üçün istifadə olunan dil və ya ifadə.

XSLT (Extensible Stylesheet Language Transformations) - XSLT (Genişlənə Bilən Stil Şablonları Dilinin Çevrilməsi): XML sənədlərini tərtib edilmiş formatlara çevirmək üçün istifadə olunan texnologiya.

# Y

YAML (YAML Ain't Markup Language) - YAML (YAML İşarələmə Dili Değildir): İnsan tərəfindən oxuna bilən məlumat formatı, strukturlu məlumatları təmsil etmək və saxlamaq üçün istifadə olunur.

Yottabyte - Yottabayt: 1,000,000,000,000,000,000,000 bayt (10^24 bayt) ölçüsündə məlumat saxlama vahidi.

Yield - Yield: Proqramlaşdırmada, xüsusilə iterativ funksiyalarda, bir nəticənin qaytarılması üçün istifadə olunan açar söz.

Y2K Problem - Y2K Problemi: 2000-ci illə əlaqəli tarix formatının bir çox proqram və sistemlərdə yalnız iki rəqəmlə (YY) göstərilməsi və bu səbəbdən baş verə biləcək səhvlər və çətinliklər.

# Z

Zero-Day Attack - Zero-Day Hücumu: Təhlükəsizlik zəifliyinin hələ istifadəçilər və ya proqram təminatçıları tərəfindən bilinmədiyi və bu zəiflikdən istifadə edən hücum.

Zero-Inflated Model - Sıfırla Dolmuş Model: Statistik modellər, xüsusilə verilənlər bazasında sıfırların təkrarlandığı hallarda istifadə olunan bir yanaşma.
